## Самосійна робота: іменовані канали та поєднання future

В наступному завдання реалізовану програму комунікації двух процесів (для обрахунку f1 та f2) через іменні канал FIFO 
з поєднання future. Також у програмі є можливість скасування процесів за нажаттям кнопки q та за кнопкою c, проте з
очікуванням підтвердження користувача

- **f1(x) = x^p** (піднесення до степеня)
- **f2(x) = a·x² + b·x + c** (квадратичний поліном)
- результат: `f1(x) * f2(x)`

Клавіші під час роботи:
- `s` — показати статус процесів
- `c` — запит на скасування (просить підтвердження `y/n`)
- `q` — вихід (скасовує без підтвердження)

---

## Структура проєкту

- `named_fifo_future.cpp` — головний процес: створення FIFO, `fork()`, відправка команд, futures, cancel/status, cleanup
- `ipc_fifo.hpp` — допоміжні утиліти: створення FIFO, читання/запис рядків, non-blocking ввід клавіш

---

## Як працює (коротко)

1. **Ввід даних**
   - Користувач вводить вхідні дані для обчислень

2. **Створення FIFO**
   - Створюються унікальні FIFO-файли`

3. **Запуск дочірніх процесів**
   - Два `fork()` -> два процеси.
   - Кожний процес відкриває свої FIFO й чекає команди запуску обрахунків (відповідних f1 та f2) або ж команди скасування

4. **Паралель через futures**
   - Батьківський процес відправляє команди в FIFO.
   - Запускає 2 задачі `std::async`, які читають відповіді дочірніх процесів та повертають результат через `std::future<double>`.

5. **Статус/скасування**
   - Окремий потік вводу читає клавіші без Enter (raw mode).
   - `c` -> запитує підтвердження `y/n`.
   - Після підтвердження головний процес надсилає дочірнім процесам `CANCEL`.

6. **Завершення**
   - Якщо все завершилось успішно: друк результатів + `STOP` процесам.
   - Якщо було скасування: друк причини + частковий статус.

7. **Прибирання**
   - Видалення дочірніх та батьківського процесів, видалення FIFO файлів з `/tmp`.
